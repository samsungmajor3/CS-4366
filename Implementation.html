Implementation Overview





To implement all the different tools and functions within our project, we created a GitHub repository to have all the code and documentation saved in a central location. The workload was divided by function, UI development, Stress_NG, and Load_Managment. The UI development was all done in Flask and HTML. The method used for the UI development had a central login page that connects to the other functions on the page via buttons. For Stress_NG, we took the stress_ng library, which can be found on Git Hub, and ran the command line functions through the FLASK API REST API to have them run on the web browser via encoded buttons. To implement the load management segment, we coded in Flask using Python to call a CPU’s process name, for example, Rack5_CPU3, to control the throttle or usage rate of the CPU by using a function call to set the subprocess CPU speed to the inputted percentage needed by the user. Building upon this, we included a function that uses a 24-hour format to schedule the rate the user wishes to set the CPU usage rate at a given time. To visualize this, HTML was used to create input boxes and buttons for these operations for the user to enter and interact with that call back to their respective functions when used. To see the CPU usage rate, a graphing function was created to read data from a CSV file to look for timestamps of 15-minute intervals and their associated CPU usage rate at that given time. As shown earlier, the function would then pass to HTML to create a graph.

Download code from git repo located at https://github.com/edward-grady/senior-capstone.git Enable a virtual environment (if operating code through local machine like with visual studio)

Windows:

     Py -3 -m venv .venv

Mac:

     (change to working directory)
     Python3 -m venv .venv
     . .venv/bin/activate
     pip install Flask schedule pandas

Run the downloaded code on local machine
If operating on webpage like replit run following command in console before running code: pip install flask

To login username and password is “admin”

To run stress_ng click the stress_ng button, then pick whether you want to test the local computer or the remote server.
In either case, you need stress-ng downloaded in your computer.

Debian packages for Ubuntu:
     sudo add-apt-repository ppa:colin-king/stress-ng
     sudo apt update
     sudo apt install stress-ng

Tarballs:
Tarballs of each version of stress-ng can be downloaded using the URL: https://github.com/ColinIanKing/stress-ng/tarball/version
where version is the relevant version number, for example: https://github.com/ColinIanKing/stress-ng/tarball/V0.13.05

Running latest stress-ng snapshot in a container:
     docker run --rm ghcr.io/colinianking/stress-ng --help
     OR

     docker run --rm colinianking/stress-ng --help

Ensure the path to the stress-ng file is usr/bin/stress-ng. You can test the local computer's CPU, memory, and I/O. Each test requires the user to input the number of CPU/ memory/ I/O workers and stress time in seconds. After that, you will click execute, and it will run the test on your computer. The result will show up below the user input section. For the remote server, you must input the Remote server IP/hostname, username, password, stress time in seconds, and number of CPU/ memory/ I/O workers, depending on the test you want to execute. You will see the result below after execution.



To run load management operations click the load management button, then input percentage of cpu usage rate to raise or lower cluster to then click set cpu button. If you want to schedule usage rate at a certain time, enter time using a 24 hour format HH:MM, then enter usage percentage and click schedule.
